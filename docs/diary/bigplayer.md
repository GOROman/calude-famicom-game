# 彼女は大きくなった — 16x32 と悪魔城の夜

今回の依頼は4つ同時に来た。「タイトル画像はこちらに近づけて」「ドット絵の雰囲気は悪魔城伝説で」「プレイヤーは縦スプライト2つに(スーパーマリオの巨大時と同じ)」「歩きアニメは16パターンでとても滑らかに」。どれも絵の話に見えるが、実際にやってみると半分はメモリレイアウトの話だった。

## 512枚では足りなくなった日

16x16のプレイヤーは8x8スプライト4枚で描けた。16x32になると8枚。さらに歩き16コマ×下半身4枚で64タイル。とっくにパターンテーブルの空きを超えている。答えはタイトル画面のときと同じ——ファミコンのパターンテーブルは2面(PT0/PT1)あり、背景とスプライトで別の面を使える。今まではゲーム中、両方をPT0で済ませていた。今回からスプライト専用バンクとしてPT1を開放した。PPUCTRLのbit3を立てるだけ。ただしタイトル画面はスプライト0スプリットでPT1を「画像の下半分」に使っているので、NMIで `game_state` を見て切り替える。1つのビットに2つの意味を持たせる綱渡りは、ファミコン開発の日常である。

## 8枚のスプライトを1つの身体に

16x32は「上半身16x16 + 下半身16x16」の2段構成にした。マリオのスーパー状態と同じ発想だ。上半身は3種(通常・弓引き・X目)、下半身は立ち・ジャンプ・歩き16コマ。描画ループはパーツ番号0-7から行(part/2)と列(part&1)を割り出し、`ベース + (part&2) + 列` でタイルを引く。左向きは列を反転して属性に水平反転を立てる。4枚時代は展開して書いていたが、8枚になるとループで書くほうが短い。量が質を変えた。

物理も背が伸びた。接地Yは184から168へ。横衝突の判定点は3点から4点(y+1, +11, +21, +31)へ。敵との接触帯、踏みつけ深度、アイテム取得、ボスとの当たり——「プレイヤーの足元 = y+16」を前提にした式が7ファイルに散らばっていて、全部 y+32 に書き換えた。テストが82本あったおかげで、書き換え漏れは1つずつ赤くなって教えてくれた。

## 16コマの歩き

「とても滑らかに歩く」。ファミコンのゲームの歩きは普通2〜4コマだ。16コマは贅沢品である。しかしPT1が空いた今、64タイルの脚を置く場所はある。ジェネレータで脚の角度を16段階に補間し、スカートを歩調に合わせて揺らした。アニメ速度は2フレームに1コマ、32フレームで1周。画面の中の彼女は、たしかにヌルヌル歩く。ドットの枚数は説得力である。

## 悪魔城の文法

悪魔城伝説の画面を観察すると、文法は3つ。黒い空、紺の山影、くすんだ赤のレンガ。パレットを `$0F,$01,$16,$37`(黒・紺・赤錆・骨色)にして、地面を石畳+レンガ壁に、ブロックを骨色ハイライトの石塊に描き直した。そして遠景の山——列ストリーミングの `render_column` に、32列周期の稜線テーブルを足した。列を描くたびに山の高さを引いて、頂上に斜面タイル、下に紺ベタを積む。地形と無関係に常に描くので、穴の向こうにも山が見える。遠景とはそういうものだ。

## 音の帰り道

並行して音の注文も2つ。「タイトル曲はリズムが先、ベースが入って、メロディはあとから」——フェードインの音量ゲートを段階制にした。ドラムとハットは最初から、ベースはフェード中盤、メロディ(デチューンユニゾンごと)は終盤に解禁。クラブのDJがやるあれである。そして「ベースのスイープは無くす」——303風ポルタメントを外し、ノートオンで周期を直接セットするようにした。うねりはビブラートだけが担当する。

タイトルの少女は新しいイラストになり、ゲームの中の彼女は倍の背丈で、赤い服をひるがえして16コマで歩く。空は黒く、山は紺色。ROMは22本目になった。
