# 番外編: URLがカセットになる日 — ステージエディタ

「ステージエディタを。Web上で動いてPagesへデプロイ。URLで保存可能。ROMデータにもなる」——要件は4つ、どれも短い。短い要件ほど設計は面白い。

## セーブデータをURLに住まわせる

このゲームのレベルは64バイトしかない。1バイトが16pxの列1本に対応し、値は0〜4の5種類。つまりステージ全体が「0〜4の数字64桁」で書ける。

64桁なら、URLに入る。

エディタはマップを編集するたびに `#0000000033000101...` とURLのハッシュを書き換える。ブックマークすればセーブ、人に送ればシェア、ブラウザバックすれば……は、さすがに効かないが、URLがそのままセーブデータになる。サーバーは何も覚えない。覚える必要がない。データが住所に住んでいる。

## ROMに焼く

「ROMデータにもなる」の実装はマーカー方式にした。アセンブラ側で `level_map` の直前に `"LVLMAP01"` という8バイトの署名を置く。エディタはベースROM(40976バイト)を取ってきて署名を検索し、その直後の64バイトを編集中のマップで上書きして、`calude-stage.nes` としてダウンロードさせる。

チェックサムのないiNESフォーマットだからできる芸当だ。カセットの時代、レベルデータはマスクROMに焼かれて工場から出荷された。いま、ブラウザの「保存」ボタンがその工場になる。

## 検証もヘッドレスで

エディタの書き出すROMが本当に動くのか。ブラウザでポチポチ確かめる代わりに、パッチ処理をNodeで再現し、改造ROMをWASMエミュレータコアに読ませて「編集で置いたブロックが実際に描画されるか」をフレームバッファで確認した。エディタを作った日から、エディタのテストも自動化されている。

デプロイはGitHub Pages。ゲームのリポジトリの `docs/` をそのまま配信し、`docs/editor/` がエディタになった。開発日誌もスクリーンショットも同じ屋根の下にいる。
