# Step 1: 空が青くなった日

2026年7月23日。空のリポジトリがひとつあった。名前は `calude-famicom-game`。タイプミスではない。「狩人(かりゅうど)」だから `Calude`。この綴りに気づいた人だけがニヤリとできる仕掛けで、このプロジェクトは始まった。

## 40976バイトの儀式

ファミコンのROMは不思議な数字でできている。16バイトのヘッダ、32KBのプログラム、8KBのキャラクタデータ。合計40976バイト。`brew install cc65` でアセンブラを入れ、リンカ設定を書き、`make` を叩く。生成された `game.nes` のサイズがちょうど40976バイトだったとき、最初の儀式は成功した。まだ何も映らないのに、もう嬉しい。

ファミコンのプログラムには「お作法」がある。電源が入ったら割り込みを切り、スタックを初期化し、PPU(画像チップ)が温まるのを vblank 2回分だけ待つ。人間で言えばラジオ体操みたいなものだ。これを省くと実機では動かない。エミュレータは優しいから動いてしまうこともある。優しさは、ときに罠だ。

## 画面をクリアする、という大仕事

「画面クリアとスプライトで左右移動とジャンプ」。文字にすれば一行の要件が、6502アセンブラでは数百行になる。ネームテーブルを空白タイルで埋め、パレットに空色($21)を流し込む。NESの色は番号で呼ぶ。$21は晴れた日の空の色。$0Fは深い夜の黒。番号で空の色を思い出せるようになったら、もう戻れない。

ジャンプは最初から少しこだわった。Y速度を8.8固定小数点で持つ。整数部と小数部を分けて、重力を毎フレーム小数部に足し込む。これだけで動きが「ぬるっ」と滑らかになる。ファミコンには小数なんてないのに、あるふりをする。プログラミングとはそういうことの積み重ねだ。

## 主人公は自分の道具でテストされる

このプロジェクトには秘密兵器がある。自作のWASMファミコンエミュレータ `cluade-famicom-emu`(こちらの綴りも意図的)だ。ブラウザで動くやつを、今回は Node.js で直接駆動することにした。

きっかけは失敗だった。ブラウザでテストしようとしたら、タブが非アクティブになるとrequestAnimationFrameが止まり、エミュレーションごと眠ってしまう。画面は真っ黒。一瞬「ROMが壊れた」と焦った。原因はただの省電力機能だった。

ならばと、WASMコアをNodeで抱え込み、`nes_frame()` を手で回すテストハーネスを書いた。ゼロページのプレイヤー座標を直接読み、「30フレーム右を押したらXは180になっているか」を機械的に検証する。ファミコンのゲームにユニットテストがある。1985年の開発者が見たら腰を抜かすだろう。でも、これが滅法便利だった。以後、すべての機能はこのテストを通ってからコミットされることになる。

## 赤いロボット、緑の狩人になる

最初のプレイヤーは赤い四角いロボットだった。ドット絵はPythonスクリプトでASCIIアートからCHRタイルに変換する方式にした。`#` が輪郭、`o` が体、`x` が肌。テキストで絵を描き、スクリプトがバイト列を吐く。

タイトルが「Calude(カリュード)」に決まった日、ロボットは緑のフードをかぶった狩人に生まれ変わった。右手には小さな弓。16x16ドットに「狩人らしさ」を詰め込むのは、俳句に季語を入れるのに似ている。制約が、かえって輪郭をくっきりさせる。

Step 1の終わり、画面には青い空と、地面に立つ小さな狩人がいた。左右に歩ける。ジャンプできる。それだけのことが、こんなに嬉しい。
