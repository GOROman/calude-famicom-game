# 512枚のタイル — イラストがタイトル画面になるまで

一枚の絵が届いた。黒い空、金色の「狩人行動」ロゴ、雪山と湖と森、そして弓を提げた狩人の少女。「この画像のイメージでタイトル画面を作成。可能な限り色など再現してください」。

ファミコンでフルスクリーンの一枚絵。制約を数えてみる。背景に同時に使えるタイルは256枚。色は4パレット×3色+共通背景色。この絵をタイル化すると、ユニークタイルは697枚あった。約2.7倍の超過である。

## 敗北の記録

最初の試みは素直だった。頻度の低いタイルから似たものへ統合して256枚へ。結果、メニューの文字が「STRRT」「OONTYNU」になった。文字は1回しか使われないタイル(頻度1)だから、真っ先に統合の犠牲になったのだ。

次に「最も似ているペアから統合」に変えたら、連鎖統合が暴走して画面全体が市松模様のノイズになった。union-findの根が代表になる実装で、統合が推移的に雪だるまになったせいだ。壊れ方が毎回違うのが画像処理の面白いところで、笑っている場合ではない。

色も迷走した。縮小にLANCZOSを使っていたため、ドット絵の輪郭が中間色に溶け、量子化が濁った。元画像は約5倍拡大のピクセルアートなのだから、NEAREST一択だった。さらにパレット割当も、貪欲法では最初に現れた茶色系ブロックが4パレットを食い潰し、緑の森と白い雪が消えた。

## 三つの武器

効いたのは三つ。**NEAREST縮小**(色が濁らない)。**ロイド式のパレット最適化**(ブロック割当→パレット再計算を収束まで反復。白・緑・青・金が戻ってきた)。**文字領域の保護**(メニュー周辺のタイルは統合距離を6倍に水増しして、統合を風景側へ誘導)。プレビューにSTART/CONTINUE/OPTIONがくっきり浮かんだときは声が出た。

## それでも256枚は足りない

「CHR-ROM足りなければサイズ拡張してOK」の一言で最終兵器が解禁された。マッパー3(CNROM)でCHRを16KBに拡張し、タイトル専用バンクを確保。それでも背景が同時に参照できるのは1パターンテーブル=256枚のまま——これはメモリではなくPPUの仕様だ。

そこで**スプライト0ヒット**を使った。画面の見えない場所(背景の裏に隠した優先度付きソリッドスプライト)を走査線136で背景と衝突させ、そのフラグを検出した瞬間にパターンテーブルをPT0からPT1へ切り替える。上半分256枚+下半分256枚=**1画面512タイル**。スーパーマリオがステータスバーの下で画面を分割していた、あの伝統芸の応用である。

NMIの中で「フラグが消えるのを待ち、点くのを待つ」——CPUが走査線を数える代わりに、PPUに教えてもらう。40年前のプログラマたちの発明は、今日も現役だ。

## 結果

エミュレータのスクリーンショットと元画像を並べる。金のロゴ、紫の山、緑の森、青い湖、狩人の少女。メニューにはスプライトの▶カーソルが灯り、上下で選べる。STARTで1-1、CONTINUEは前回のステージから。ミスするとBGMが止まり、ジングルだけが響くようにもなった。

一枚の絵が、49,168バイトのカセットに収まった。
