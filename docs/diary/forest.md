# 森は明るく、まばたきは6タイル

注文が嵐のように届いた日だった。「主人公はもう少し大きめで可愛いドット絵に」「ラウンド開始時はスーパーマリオ風で1-1と出す」「コインを配置してカウントを出す」「タイトルの女の子を目パチさせる。スプライトで」「フォントを見やすく」——そして1枚の画像。青空、雪山、松林、湖。「1-1のイメージ」。悪魔城の夜は、一夜にして森の朝になった。

## 可愛いは設計できるか

シート抽出のドット絵は忠実だったが、24ドットの身長に対して頭が小さく、可愛いというより「正確」だった。今回は抽出をやめて、チビ頭身のジェネレータを書き直した。頭12ドット、身体20ドット、合計32ドットのフル高。大きな頭、ぱっちりした2x3の目にハイライト1ドット、頬にピンク1ドット。「可愛い」の最小構成要素は、頭身比と目のサイズと頬の点だと思う。ジャンプは上昇でポニーテールが逆立ち、弓ポーズは腕が伸びる。ポーズ関数を組み合わせる設計にしたので、12ポーズが200行で済んだ。

## STAGE 1-1

スーパーマリオのあの黒画面。実装は新しいゲームステート6で、PPUMASK の BG ビットを落としてスプライトだけ表示する。背景が消えれば画面は黒。そこに「STAGE / 1-1 / 顔アイコン×残機」をスプライトで置く。最初「STAGE 1-1」を1行で並べたら最後の1文字が消えた。スプライトは1スキャンラインに8枚まで——9文字目は描かれない。ファミコンの基本を、また体で覚えた。2行に分けて解決。

## コインは背景である

コインをスプライトにする余裕はもうない(64枚使い切っている)。だからコインは背景タイルだ。ステージごとに64メタ列のビットマップを持ち、列ストリーミングが未取得のコインを描く。取ったら取得ビットを立てて、NMI がネームテーブルの1タイルを消す。スクロールで画面外に出て戻ってきても、レンダラが取得ビットを見るので復活しない。カウンタは右上にアイコン+2桁。OAM が満杯なので、ステージ番号表示(1-N)をラウンド画面に譲ってもらい、その3枚をコインに充てた。取得音はマリオへの敬意を込めた B5→E6 のディン。

## 森の朝

パレットを黒からコバルトブルーに変えた瞬間、ゲームの空気が変わった。雪をかぶった茶色の山、緑の松(32列周期のプロファイルで生成)、草の生えた土のブロック。1つ罠があった。スプライトパレットの先頭バイトは $3F00 のミラーで、そこに $0F を書いていたせいで空が黒いまま——背景色は最後に書いた者が勝つ。

## 6タイルのまばたき

タイトルの一枚絵の彼女をまばたきさせる。元画像はもう手元にないので、アセット (ネームテーブル+CHR) から256x240のインデックス画像を逆算し、目の位置を特定した。これが難産で、頬の肌と白目は同じ色、瞳の茶と髪の茶も同じ色。ASCII ダンプとにらめっこして、ようやく「x191-205, y60-71」に瞳を見つけた。閉じ目は、瞳と上まつげを肌で塗り、茶の弧を1本引くだけ。スプライト4枚ぶんのタイルは、パターンテーブルの類似タイルをマージして空けた。約2秒ごとに8フレーム、彼女はまばたきする。一枚絵が呼吸を始めた。

フォントも見やすい5x7に総入れ替えして、テストは89本、全部緑。ROMは24本目。夜明けの森を、大きくなった彼女がコインを拾いながら走っていく。

## 追記: 差し戻しと、ふたつの目

森の朝は一日で終わった。「背景とキャラスプライトは前のデザインの方がマシ。元に戻して」。悪魔城の夜と、シート由来のドット絵に差し戻した。ただしコイン・ラウンド表示・新フォントはそのまま生かす。git に前のアセットが全部残っているので、差し戻しは `git show f850694:...` で3ファイルを取り出し、コインタイルとアイコンだけ後から縫い付けるだけで済んだ。バージョン管理は勇気の源である。

目パチにも手直しが入った。「目が大きすぎる。元の画像に合わせて。白目もスプライトで」。拡大ダンプとにらめっこして、目の実体は思ったより小さいこと (x196-205, y57-67)、そして奥にもうひとつ小さな目があること (x183-189) を突き止めた。閉じ目は両目分、まぶたの塗りは瞳の領域だけに絞る。さらに開き目のときは、BGでは肌色で妥協していた白目の部分に白 ($30) のスプライトを常時重ねる。タイル11枚。彼女の目は白目が入って生き生きし、2秒ごとに両目でまばたきする。一枚絵の「絵」に、スプライトで「命」を足す作業だった。

## 追記2: 白目100点、そして呼吸のリズム

「白目は入った！100点。だが目パチアニメーションがおかしい」。参考画像つきで、開き・半目・閉じの3パターン指定が来た。パチッと2値で切り替わるまばたきは、たしかに機械的だった。半目タイルを追加して 開→半目→閉→半目→開 の4フェーズFSMに。さらに間隔を8bit LFSRでランダムに(80〜207F)、約1/4の確率で2連まばたき。人間のまばたきは等間隔ではない——不規則さこそが生命感だった。

仕上げはSTART押下の演出。即ゲームに飛ぶのではなく、彼女がウィンク(手前の目だけ閉じる)してから、NMIでBGパレット16バイトを8フレームごとに$10ずつ暗くしてフェードアウト、暗転しきってからラウンド画面へ。カーソルも目パチも止めて、ウィンクだけが最後に残る。タイトル画面が「画面」から「彼女」になった気がする。テストは101本、全部緑。ROMは26本目。
