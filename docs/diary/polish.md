# 磨きの日 — スコア、二曲目、そしてデチューン

ロードマップは完走した。ここからは磨きだ。

## スコアは10進で数える

スコアを付けた。矢で100点、踏みつけで200点(勇気の分だけ高い)、アイテム500点、クリア1000点。6502には10進モードがある——が、ファミコンのCPU(2A03)はそれが**無効化されている**という有名な話があり、スコアは自前の桁配列で持つことにした。100点単位の4桁+固定の「00」。桁あふれは上の桁へ手動で繰り上げる。小学生の筆算をアセンブラで書いている気分になる。999900点でカンスト。そこまで狩れる者がいたら会ってみたい。

## タイトルの曲、ゲームの曲

「タイトルは今の感じの曲で、1-1の曲は元に戻す」。コード進行(Am→F→C→G)はタイトル向きの風格があり、ゲーム中は元のAmグルーヴのほうが走れる——言われてみればその通りで、曲を2つに分けた。ドライバは `game_state` を見てパターンテーブルを切り替えるだけ。同じ楽器(808、303、パルス2本)で、場面によって別の顔をする。

## デチューンという魔法

そして「タイトルの矩形波はデチューンで広がりを(ドラクエ2のように)」。すぎやまこういち先生のアレだ。2本の矩形波に**同じメロディを、周期を1だけずらして**重ねる。数セントのピッチ差が毎秒数回のうねりを生み、たった2音がストリングスのように広がる。ファミコン音楽の最重要テクニックのひとつで、実装はわずか `adc #1`。3バイトで手に入る豊かさとしては、ゲーム史上最高のコストパフォーマンスだと思う。

検証も面白かった。520フレームぶんレジスタを записать、SQ2の周期が常にSQ1+1であることを確認(一致率100%)。ゲーム曲側はエコーがきっかり16フレーム遅れ(一致率100%)。耳で聴く前に、数字が「良い音」を保証してくれる。

## エディタも4面に

ステージが4つになったので、エディタにも1-1〜1-4のタブを付けた。デフォルトのマップはハードコードをやめて、**base.nesのLVLMAP01マーカーから直接読む**ようにした。ゲームを更新すればエディタの初期値も追従する。データの出どころは一箇所であるべし——ROMがそのまま真実の源(single source of truth)になった。
