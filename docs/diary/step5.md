# Step 5: ROMの中の TR-808

「TR-808風のサウンドをDMCで再生。ハイハットはNOI、ベースはTRI。LFOやエンベロープを駆使する」——発注書はDTM用語で書かれていた。ファミコンのAPU(音源チップ)は5チャンネル。矩形波2、三角波、ノイズ、そしてDMC。今回はこのうち3つで、1980年のリズムマシンを1983年のゲーム機に召喚する。

## キックは波形ごと焼く

808のバスドラムの正体は「ピッチが落ちていくサイン波」だ。110Hzから42Hzへ指数カーブで沈み込みながら減衰する。これをPythonで合成し、**DPCM**(1ビットのデルタ変調)にエンコードしてROMの$C000に焼いた。929バイト。カセットの中に、キックの波形そのものが入っている。

DMCはファミコン唯一のサンプラーだ。1ビット/サンプルという極限の圧縮なので、デルタ変調特有のザラつきが乗る。ところがこのザラつきが、808キックの「ブォン」にローファイな味を足す。劣化が音色になる稀有な例だ。スネアも同じ手法で作った。ノイズと180Hzのボディを混ぜて625バイト。

DMCは1本しかないので、キックとスネアが同じステップに来たらスネアを優先する。808の実機でも全部同時には鳴らせない音があった。制約は再現の一部である。

## ハイハットはエンベロープで刻む

ノイズチャンネルには音量エンベロープをソフトウェアで書いた。トリガーで音量10からスタートし、毎フレーム3ずつ減らす——これがクローズハット。減衰1にするとオープンハットになる。同じノイズ源でも、減衰カーブが楽器を分ける。「チッ」と「シャーン」の違いは、音色ではなく時間の形だ。

## ベースは揺れる三角波

三角波でベースラインを鳴らす。A1(55Hz)を中心にしたパターンをシーケンサが16分で刻む。そのままだと機械的すぎるので、**LFO**を載せた。16エントリのサインテーブルで周期レジスタを±2揺らす。周期32フレーム≈1.9Hzのビブラート。テストのログに周期値が「248,247,246,249,250…」と並んだとき、揺れが数字で見えた。

## シーケンサは8フレームで1歩

ドライバの心臓は16ステップのシーケンサだ。1ステップ=8フレーム、つまり約112BPMの16分音符。ドラムパターンは1ステップ1バイトのビットマスク(bit0=キック、bit1=スネア、bit2=クローズ、bit3=オープン)。4つ打ちのキック、バックビートのスネア、8分のオープンハット。1小節ぶんの音楽が、たった16バイトに収まっている。

## 無音のバグ

ヘッドレステストで音声を検証しようとしたら、サンプル数が0だった。レジスタは正しく動いているのに、音がない。原因はエミュレータの `nes_init(サンプルレート)` を引数なしで呼んでいたこと。レート0Hz——「1秒間に0回サンプリングする」世界では、どんな名曲も無音だ。44100を渡したら、RMS 0.43の元気なグルーヴが返ってきた。

出来上がった音は `docs/bgm_sample.wav` に4秒だけ録音してある。ROMの中の小さな808が、今日も4つ打ちを刻んでいる。
